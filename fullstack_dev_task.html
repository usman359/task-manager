What you'll build

A lightweight task manager with a REST API backend and a simple frontend. The app should allow users to manage tasks with basic CRUD operations, filtering, and real-time-like status updates.

Backend REST API
  • Build a REST API with endpoints: GET /tasks, POST /tasks, PATCH /tasks/:id, DELETE /tasks/:id
  • Each task has: id, title, description, status (todo / in-progress / done), priority (low / medium / high), createdAt
  • Support query param filtering: GET /tasks?status=todo&priority=high
  • Validate inputs and return proper HTTP status codes with error messages
  • Persist data in any database (SQLite is fine) — no in-memory only
Frontend UI
  • Display all tasks in a list; show title, priority badge, and status
  • Filter tasks by status and/or priority without a page reload
  • Form to create a new task with client-side validation
  • Inline status update (e.g. dropdown or button) that calls the API and reflects immediately
  • Delete a task with a confirmation prompt
  • Basic responsive layout (must work on mobile width)
Bonus Optional extras (only if time allows)
  • Drag-and-drop to reorder tasks within a status column
  • Simple auth — a hardcoded API key passed in a request header, validated on every route
  • Unit test for at least one API endpoint

Tech stack

You may use any of the following. Pick what you know best — the task is the same regardless.

  • Backend: Node.js (Express / Fastify / Hono), Python (FastAPI / Flask), or Go
  • Frontend: React, Vue, Svelte, or plain HTML/JS — your choice
  • Database: SQLite, PostgreSQL, or MongoDB
  • Styling: Any CSS framework or vanilla CSS
